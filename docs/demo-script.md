# 🎯 CASPER AR Assistant — Демо-сценарий для жюри

**Команда:** «Команда которая объективно лучше всех, но судьи этого не замечают»
**Хакатон:** Цифровой Вызов от Casper AI, Нижний Новгород 2026
**Время демо:** 3–5 минут

---

## Подготовка (до выхода к жюри)

### 1. Запуск сервера
```cmd
cd backend
.venv\Scripts\activate.bat
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 2. Запуск ngrok (для доступа с телефонов жюри)
```cmd
rem В отдельном окне:
ngrok http 8000
```
ngrok выдаст URL вида `https://abc123.ngrok-free.app` — **запиши его**.
Именно этот URL даёшь жюри для открытия на своих телефонах.

> ⚠️ Важно: ngrok нужен потому что камера в браузере работает только по HTTPS.
> На localhost камера работает только в Chrome/Edge с флагом, на телефонах — только HTTPS.

### 3. Сброс БД (чистый старт для жюри)
```cmd
python reset_db.py
```

### 4. QR-код для жюри (опционально)
Открой https://qr-code-generator.com/ → вставь ngrok URL → распечатай или покажи на экране.
Жюри сканирует QR → попадает прямо в приложение.

### 5. Маркеры
Открой **https://[твой-ngrok-url]/markers** и распечатай (или открой на планшете):

| Маркер | ID | Роль | Где разместить |
|--------|----|------|----------------|
| 🚪 Входная зона | 0 | Квест «Первый шаг» | На двери / входе |
| 🛑 Аварийный стоп | 1 | Квест «Аварийная остановка» | На стене / столе |
| 🧯 Огнетушитель | 4 | Квест «Где огнетушитель?» | На стене / столе |
| ⚙️ Рабочий станок | 2 | Квест «Знакомство со станком» | На ноутбуке / принтере |
| 🔧 Зона инструмента | 3 | Квест «Финальный осмотр» | На столе |
| ⛑ Каска (СИЗ) | 10 | Safety Check | Держи у лба |
| 🦺 Жилет (СИЗ) | 11 | Safety Check | Держи у груди |

## Сценарий демо с телефона жюри (3 минуты)

### Фаза 0: Подключение жюри (30 сек)
1. Покажи жюри QR-код или продиктуй ngrok URL
2. *«Откройте на своём телефоне — приложение не нужно устанавливать»*
3. Жюри открывает `https://[ngrok-url]` в браузере телефона
4. Если браузер просит разрешение на камеру — нажать «Разрешить»

### Фаза 1: Вход и Safety Check (40 сек)
1. Жюри вводит никнейм (например `Judge_1`) → «Начать миссию»
2. **Автоматически** открывается Safety Check — его нельзя пропустить
3. *«Первое что видит новый сотрудник — обязательная проверка снаряжения»*
4. Поднеси маркер **ID 10** к камере телефона жюри → каска ✅
5. Поднеси маркер **ID 11** → жилет ✅
6. Кнопка «Начать смену» зеленеет → жюри нажимает
7. *«Только после проверки СИЗ — доступ к рабочему экрану»*

### Фаза 2: Онбординг и маскот (20 сек)
1. Появляется 5-шаговый онбординг — жюри листает
2. После закрытия — маскот КАСПЕР приветствует голосом
3. *«Маскот снижает стресс первых дней — есть режим только текст и текст с озвучкой»*

### Фаза 3: AR-детекция — ГЛАВНОЕ (60 сек)
1. Жюри нажимает 🎯 → выбирает «Первый шаг» → подтверждает
2. *«Наводим камеру телефона на маркер входной зоны»*
3. Жюри наводит свой телефон на маркер **ID 0** (🚪)
4. Маркер обводится **зелёным полигоном** прямо на экране телефона
5. Появляется кнопка «✅ Засчитать» — жюри нажимает сам
6. *«+50 XP, уровень вырос — маскот поздравляет»*

### Фаза 4: Энциклопедия (20 сек)
1. После сканирования появляется фиолетовый тост «📖 Узнай больше»
2. Жюри нажимает — карточка объекта открывается сразу на нужном объекте
3. *«Сотрудник сразу узнаёт правила работы с объектом который только что нашёл»*
4. Закрыть — квест уже засчитан, второй раз сканировать не нужно

### Фаза 5: Профиль и лидерборд (20 сек)
1. Жюри нажимает 👤 → видит свой профиль с набранными XP
2. Показать статистику: квесты, сканирования, ачивки
3. *«Несколько человек из жюри — уже лидерборд. Добавьте коллег»*

### Финальный тезис (15 сек)
> *«Зашёл → прошёл safety check → навёл камеру → получил подсказку → вырос в уровне. Никаких приложений, никакого сложного оборудования — только телефон и ссылка»*

## Ответы на вопросы жюри

**«Как работает распознавание?»**
> OpenCV ArUco — маркеры распознаются мгновенно без нейросети. Для реального внедрения маркеры заменяются на fine-tuned YOLOv8 модель, обученную на конкретном оборудовании предприятия.

**«Почему геймификация?»**
> Исследования показывают: молодёжь (18–25 лет) воспринимает обучение через игровые механики на 40% эффективнее. Квесты создают понятную структуру первых дней, XP и ачивки дают мгновенную обратную связь.

**«Как внедрить на реальном заводе?»**
> 1. Обучить YOLOv8 на фото оборудования конкретного предприятия (1–2 дня).
> 2. Распечатать маркеры и разместить как таблички (15 минут).
> 3. Развернуть сервер в локальной сети завода — смартфон подключается через Wi-Fi.
> 4. Добавить контент квестов через YAML-файл без программирования.

**«Что с безопасностью данных?»**
> Вся обработка — на сервере завода. Видеопоток не покидает локальную сеть. Камера используется только в момент сканирования.

**«Сколько стоит внедрение?»**
> Только смартфон/планшет у сотрудника (часто уже есть) + недорогой сервер. Не нужны специальные очки AR или дорогое оборудование.

## Технический стек (для технической части жюри)

```
Backend:   Python 3.13, FastAPI, SQLAlchemy (SQLite → PostgreSQL)
CV:        OpenCV ArUco, YOLOv8n (ultralytics)
Transport: REST API + WebSocket real-time
Frontend:  Vanilla JS, Canvas 2D AR overlay
Auth:      JWT (python-jose)
Game:      XP engine, quest chains, achievements system
Access:    ngrok HTTPS tunnel → любой телефон без установки
```

## Резервный план (если что-то не работает)

- ngrok упал → перезапустить `ngrok http 8000`, новый URL
- Камера не работает → проверить что открыт именно HTTPS (ngrok URL)
- Маркер не детектится → улучши освещение, держи ровнее, расстояние 20–50 см
- Сервер упал → `uvicorn app.main:app --reload` в backend/
- БД сломалась → `python reset_db.py`
